  <Accordion title="excel_utils.py — Lectura y limpieza de Excel">
    Responsable de leer el archivo Excel o CSV de entrada, detectar las hojas relevantes y dejar el `DataFrame` limpio y normalizado para el resto del sistema.

    ### `cargar_excel_crudo(file_path)`

    Lee el archivo Excel buscando hojas de **entrantes**, **salientes** y **DATOS**. Si no detecta hojas específicas, lee la primera hoja como genérica.

    ```python
    def cargar_excel_crudo(file_path: str) -> tuple[pd.DataFrame | None, str | None]:
        ...
    ```

    | Parámetro | Tipo | Descripción |
    |---|---|---|
    | `file_path` | `str` | Ruta absoluta al archivo `.xlsx`, `.xls` o `.csv` |

    **Retorna:** `(DataFrame, None)` si la lectura fue exitosa, o `(None, mensaje_error)` si falló.

    La función aplica estas transformaciones automáticas:
    - Normaliza los nombres de columna a minúsculas sin espacios.
    - Elimina filas completamente vacías.
    - Asigna el valor de `tipo_llamada` según la hoja de origen (`entrante`, `saliente`, `DATOS`).
    - Para la hoja de DATOS, mapea columnas técnicas (`fecha_trafico` → `fecha_hora`, `latitud` → `latitud_n`, etc.).

    ### `procesar_dataframe_con_mapeo(df, mapping)`

    Aplica limpieza profunda al `DataFrame` crudo: normalización de teléfonos, parseo de fechas y corrección de coordenadas.

    ```python
    def procesar_dataframe_con_mapeo(
        df: pd.DataFrame,
        mapping: dict[str, str] | None
    ) -> pd.DataFrame | None:
        ...
    ```

    | Parámetro | Tipo | Descripción |
    |---|---|---|
    | `df` | `pd.DataFrame` | DataFrame crudo resultante de `cargar_excel_crudo()` |
    | `mapping` | `dict` o `None` | Mapa `{columna_interna: columna_excel}` producido por el diálogo `ColumnMapperDialog` |

    El procesamiento sigue este orden:

    <Steps>
      <Step title="Aplicar mapeo de columnas">
        Renombra las columnas del Excel a los nombres internos del sistema (`originador`, `receptor`, `fecha_hora`, etc.).
      </Step>
      <Step title="Normalizar teléfonos">
        Aplica `normalizar_numero_colombia()` sobre las columnas `originador` y `receptor`.
      </Step>
      <Step title="Parsear fechas">
        Invoca `robust_date_parser()` para convertir cada valor de `fecha_hora` al tipo `datetime`, manejando formatos seriales de Excel y texto variado.
      </Step>
      <Step title="Corregir coordenadas">
        Detecta y corrige errores de escala (p. ej. `7375060` → `7.375060`), reemplaza comas por puntos y elimina valores inválidos (`0`, `?`, `nan`).
      </Step>
    </Steps>

    ### `robust_date_parser(value)`

    Motor interno de parseo de fechas. Maneja en orden:
    1. Valores nulos o cadenas vacías → `pd.NaT`
    2. Números decimales en formato serial de Excel (rango 32874–51138)
    3. Formatos de texto específicos: `%Y/%m/%d %H:%M:%S`, `%d/%m/%Y %H:%M:%S`, `%Y-%m-%d %H:%M:%S`, etc.
    4. Parseo automático de pandas como último recurso
  </Accordion>

  <Accordion title="geo_utils.py — Generación de mapas">
    Genera los tres tipos de mapas interactivos que se incrustan en el informe HTML. Usa Folium para el mapa de clusters y el mapa de calor, y Plotly para el timeline de recorrido de datos.

    ### `generar_mapa_agrupado(df, output_path, nombres_asignados=None)`

    Mapa Folium de clusters separado por tipo de llamada (entrantes en azul, salientes en verde, datos en morado).

    ```python
    def generar_mapa_agrupado(
        df: pd.DataFrame,
        output_path: str,
        nombres_asignados: dict[str, str] | None = None
    ) -> None:
        ...
    ```

    - Requiere columnas `latitud_n` y `longitud_w` con valores numéricos válidos.
    - Genera tres capas `MarkerCluster` independientes con control de capas.
    - Los marcadores muestran en el popup: número (con alias si está asignado), fecha/hora y tipo de llamada.
    - Guarda el resultado en `output_path` como archivo HTML autocontenido.

    ### `generar_mapa_rutas(df, output_path, nombres_asignados=None)`

    Timeline animado (Plotly) del recorrido cronológico de registros de datos de internet. Se usa **exclusivamente** con las filas de tipo `DATOS`, nunca con llamadas de voz.

    ```python
    def generar_mapa_rutas(
        df: pd.DataFrame,
        output_path: str,
        nombres_asignados: dict[str, str] | None = None
    ) -> None:
        ...
    ```

    - Ordena los registros por `fecha_hora` antes de animar.
    - Muestra un slider de tiempo con el formato `YYYY-MM-DD HH:MM:SS`.
    - El tooltip tipo card muestra: nombre de antena (o ID de celda), fecha, hora y coordenadas.
    - Sin autoplay: el usuario controla la reproducción con los botones ▶ / ⏸.

    ### `generar_mapa_calor(df, output_path)`

    Mapa de calor Folium que visualiza la densidad de actividad por zona geográfica.

    ```python
    def generar_mapa_calor(
        df: pd.DataFrame,
        output_path: str
    ) -> None:
        ...
    ```

    - Radio del punto de calor: `15` px; difuminado: `20` px (según `MAP_HEATMAP_RADIUS` y `MAP_HEATMAP_BLUR` en `constants.py`).
    - Usa fondo OpenStreetMap.
    - Filtra automáticamente filas sin coordenadas válidas mediante `_limpiar_coordenadas()`.

  <Accordion title="report_generator.py — Generación del informe HTML">
    Orquesta la construcción completa del informe: mapas, gráficos, adjuntos y renderizado con Jinja2.

    ### `generar_informe_html(...)`

    ```python
    def generar_informe_html(
        df: pd.DataFrame,
        nombre_informe: str,
        incluir_membrete: bool = False,
        logo_path: str | None = None,
        lista_adjuntos: list[dict] | None = None,
        nombres_asignados: dict[str, str] | None = None,
        datos_generales: dict | None = None
    ) -> str:
        ...
    ```

    **Retorna** la ruta absoluta a la carpeta del informe generado (`output/<nombre_informe>/`).

    El proceso interno sigue estos pasos:

    <Steps>
      <Step title="Separación de registros">
        Divide el DataFrame en `df_llamadas` (voz) y `df_datos` (internet). Las estadísticas y tablas solo usan `df_llamadas`. El mapa de recorrido usa exclusivamente `df_datos`.
      </Step>
      <Step title="Copiar adjuntos PDF">
        Copia cada PDF de `lista_adjuntos` a `output/<nombre>/data/` y registra su categoría y ruta relativa.
      </Step>
      <Step title="Copiar recursos estáticos">
        Copia `interactive_charts.js`, `interactive_maps.js`, `logo.png` e `info.png` al subdirectorio `static/` del informe para hacerlo portable.
      </Step>
      <Step title="Generar mapas">
        Invoca `generar_mapa_agrupado()` y `generar_mapa_calor()` sobre `df_llamadas`, y `generar_mapa_rutas()` sobre `df_datos` (solo si este no está vacío).
      </Step>
      <Step title="Calcular estadísticas y geografía">
        Calcula top entrantes, top salientes, promedio de llamadas y, para cada registro, infiere departamento y municipio con `obtener_ubicacion_completa()`.
      </Step>
      <Step title="Renderizar plantilla Jinja2">
        Renderiza `templates/report_template.html` con todas las variables calculadas y escribe el HTML resultante en `reports/informe_llamadas.html`.
      </Step>
    </Steps>

    Implementa el Plan B de geocodificación: cruzar el nombre técnico de la antena contra la base de datos `celdas.csv` para obtener sus coordenadas exactas.

    ### Clase `CellGeocoder`

    ```python
    class CellGeocoder:
        def __init__(self, db_path: str):
            ...
    ```

    Al instanciarse, carga y procesa automáticamente el CSV de celdas mediante `cargar_base_datos()`.

    #### `limpiar_nombre_sitio(nombre)`

    Elimina sufijos técnicos del nombre de la antena para encontrar el nombre raíz del sitio físico.

    ```python
    def limpiar_nombre_sitio(self, nombre: str) -> str:
        ...
    ```

    ```text
    Entrada:  "ANT.BARBOSA-2_R1"
    Proceso:  regex r'_([A-Z0-9]{1,4}|LTE|UMTS|GSM)$' aplicado dos veces
    Salida:   "ANT.BARBOSA-2"
    ```

    #### `cargar_base_datos()`

    Carga el CSV intentando primero separador `;` (latin-1) y luego `,`. Agrupa por nombre raíz promediando las coordenadas de las celdas del mismo sitio físico.

    #### `buscar_coordenadas(df_cdr, col_nombre_celda)`

    Realiza un LEFT JOIN entre el DataFrame de CDR y la base de celdas indexada.

    ```python
    def buscar_coordenadas(
        self,
        df_cdr: pd.DataFrame,
        col_nombre_celda: str
    ) -> pd.DataFrame:
        ...
    ```

    - Rellena `latitud_n` y `longitud_w` donde estaban vacíos, sin sobreescribir coordenadas GPS nativas (Plan A).
    - Imprime el número de registros ubicados exitosamente.

  <Accordion title="colombia_data.py — Inferencia geográfica por municipio">
    Implementa el Plan C de geocodificación: inferir coordenadas a partir del nombre del municipio detectado en el texto de la celda, y asignar departamento y municipio a coordenadas conocidas.

    ### `obtener_ubicacion_completa(lat, lon)`

    Recibe coordenadas y devuelve el departamento y municipio más cercanos usando la fórmula de Haversine.

    ```python
    def obtener_ubicacion_completa(
        lat: float,
        lon: float
    ) -> tuple[str, str]:
        ...
    ```

    **Retorna:** `(departamento, municipio)` — por ejemplo: `("Antioquia", "Barbosa")`

    La búsqueda se optimiza filtrando primero un recuadro de ±1° alrededor del punto antes de calcular distancias exactas.

    ### `inferir_municipio_y_coords(nombre_celda)`

    Busca el nombre de un municipio dentro del texto técnico de la celda.

    ```python
    def inferir_municipio_y_coords(
        nombre_celda: str
    ) -> tuple[str | None, float | None, float | None]:
        ...
    ```

    **Retorna:** `(municipio, latitud, longitud)` si se encuentra coincidencia, o `(None, None, None)`.

    ```text
    Entrada:  "ANT.MEDELLIN-2"
    Búsqueda: compara texto normalizado (sin tildes) con cada municipio de la BD
    Salida:   ("Medellín", 6.244203, -75.581211)
    ```

    <Note>
      Se ignoran municipios con nombres de menos de 4 caracteres para evitar falsos positivos (ej: "Ica", "Une").
    </Note>

    ### `calcular_distancia(lat1, lon1, lat2, lon2)`

    Define las estructuras de datos mediante `dataclasses` de Python.

    ### `Llamada`

    Modelo que representa un registro CDR individual.

    ```python
    @dataclass
    class Llamada:
        originador:  str
        receptor:    str
        duracion:    int
        fecha_hora:  datetime
        latitud_n:   float = None
        longitud_w:  float = None
    ```

    El método `__post_init__` realiza validaciones automáticas al instanciar:
    - Convierte `fecha_hora` desde string (`"%Y-%m-%d %H:%M:%S"`) si es necesario.
    - Convierte `duracion` a `int`.
    - Valida y corrige coordenadas mediante `validar_coordenada()`.

    #### `validar_coordenada(valor)`

    Corrige errores de escala típicos en datos de Excel: si el valor absoluto supera 180, lo divide entre 10000.

    ### `ResumenLlamadas`

    Modelo para estadísticas generales del análisis.

