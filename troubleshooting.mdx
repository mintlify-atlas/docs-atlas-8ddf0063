    **SĂ­ntoma:** La aplicaciĂłn no muestra datos despuĂŠs de seleccionar el archivo, o aparece un error en el panel de logs.

    **Causas posibles y soluciones:**

    **1. El archivo estĂĄ abierto en Excel**

    Cierra el archivo en Excel antes de cargarlo. `openpyxl` no puede leer archivos con bloqueo de escritura activo.

    **2. El archivo estĂĄ en formato `.csv` con codificaciĂłn incorrecta**

    La funciĂłn `cargar_excel_crudo()` lee Excel con `pd.read_excel()`. Para CSV, asegĂşrate de que el archivo use UTF-8 o latin-1. Si el archivo usa otro encoding, conviĂŠrtelo primero:

    ```bash

    ```

    **3. Las hojas no tienen nombres reconocibles**

    El sistema busca hojas que contengan `entrant`, `incoming`, `salient`, `outgoing` o `dato` en su nombre. Si tus hojas tienen nombres diferentes (ej: `Hoja1`, `Llamadas`), el sistema lee la primera hoja como genĂŠrica y asigna tipo `desconocido`. Renombra las hojas en Excel, o usa el asistente de mapeo de columnas para asignar el campo `tipo_llamada`.

    **4. Todas las fechas resultan invĂĄlidas**

    Si el log muestra `â Error: Todas las fechas fueron invĂĄlidas`, el formato de fecha de tu archivo no estĂĄ entre los soportados. Los formatos aceptados son:

    ```text

    %Y/%m/%d %H:%M       â 2025/10/08 12:04
    %d/%m/%Y %H:%M:%S    â 08/10/2025 12:04:47
    %d-%m-%Y %H:%M:%S    â 08-10-2025 12:04:47

    %Y%m%d %H%M%S        â 20251008 120447
    Formato serial Excel  â nĂşmero decimal entre 32874 y 51138
    ```
  </Accordion>

    **3. El archivo tiene columnas duplicadas**
